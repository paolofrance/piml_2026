# Lagrangian Neural Networks (LNN)

## Method

**Reference:** Cranmer, Greydanus, Hoyer, Battaglia, Spergel & Ho, *Lagrangian Neural Networks*, ICLR 2020 workshop.

### Core idea

Instead of learning the acceleration directly `(q, q̇) → q̈`, an LNN learns a scalar **Lagrangian**:

```
L_θ(q, q̇) ∈ ℝ
```

and derives the equations of motion analytically through the **Euler-Lagrange equations**:

```
d/dt (∂L/∂q̇) − ∂L/∂q = 0
```

Expanding this gives:

```
M(q, q̇) q̈ = ∂L/∂q − C(q, q̇) q̇
```

where the **generalised mass matrix** M and **Coriolis vector** C are computed by differentiating the network twice with `torch.autograd`.

### Why energy conservation emerges

Any scalar function of `(q, q̇)` that satisfies the Euler-Lagrange equations will produce a vector field that is symplectic (area-preserving in phase space). This means the total mechanical energy `E = T + V` is a conserved quantity along any trajectory generated by the model — not because it is explicitly constrained, but because Lagrangian mechanics implies it.

In contrast, a VanillaNN that learns `(q, q̇) → q̈` has no such guarantee: nothing prevents it from predicting accelerations that drain or inject energy.

### Training

LNN is trained as a **dynamics model**: given a dataset of `(q, q̇, q̈)` tuples collected from the true system, the loss is:

```
L = MSE(q̈_predicted, q̈_true)
```

Rollout is done by integrating the learned acceleration with RK4, not by unrolling through the network during training.

### Limitation: no guarantee on M

The mass matrix M is computed as the Hessian of the scalar network `L_θ` w.r.t. `q̇`. Nothing in the architecture forces this Hessian to be symmetric positive-definite (SPD), which is a physical requirement. For well-trained networks with Softplus activations this is approximately satisfied, but it can fail for ill-conditioned problems. DeLaN (see `delan/`) addresses this with an explicit Cholesky parameterisation.

### Extension to dissipative systems: LNN-F

Adding a **Rayleigh dissipation function** `D(q, q̇) = ½ q̇ᵀ B(q) q̇` modifies the EOM:

```
M(q, q̇) q̈ = ∂L/∂q − C(q, q̇) q̇ − B(q) q̇
```

`B(q) > 0` is enforced via a `softplus` output from a separate network. This guarantees `dE/dt = −q̇ᵀ B q̇ ≤ 0`: energy can only decrease, which is the physical requirement for a dissipative system.

## Examples

### `lnn_vs_vanilla.py` — Simple pendulum, conservative (data-scarce regime)

**System:** `θ̈ = −(g/L) sin(θ)`  (g=9.81, L=1)

**Setup:** 20 samples from `t ∈ [0, 3 s]` with θ₀=2.5 rad (near the separatrix — highly nonlinear trajectory). Evaluation rollout over `[0, 12 s]`.

**Models:**
- `LNN`       — Lagrangian network, energy conserved by construction
- `VanillaNN` — unstructured `(θ, θ̇) → θ̈` regression

**Key result:** with only 20 data points from a near-separatrix trajectory, VanillaNN fails to generalise. LNN's Lagrangian structure constrains the rollout to stay on the correct energy shell far beyond the training window. The energy plot is the decisive diagnostic: VanillaNN drifts, LNN stays flat.

---

### `lnn_vs_delan.py` — Spring pendulum 2-DOF: LNN vs DeLaN vs VanillaNN

**System:** spring pendulum in polar coordinates `(r, θ)` — same system as `examples/delan/delan_vs_vanilla.py`.

**Setup:** 80 samples from `t ∈ [0, 3 s]`. Evaluation rollout over `[0, 8 s]` in Cartesian `(x, y)`.

**Models:**
- `DeLaN`     — SPD mass matrix (Cholesky) + non-negative potential (softplus)
- `LNN`       — unstructured scalar Lagrangian
- `VanillaNN` — unstructured `(q, q̇) → q̈` regression

**Key result:** all three models achieve comparable interpolation accuracy. In extrapolation, both Lagrangian methods outperform VanillaNN. DeLaN further outperforms the vanilla LNN — the guaranteed positive-definiteness of its mass matrix prevents the numerical instabilities that arise when the LNN's implicit mass matrix becomes near-singular on the coupled 2-DOF system. The energy panel makes the hierarchy explicit: DeLaN < LNN < VanillaNN in energy drift.

*Use after `lnn_vs_vanilla.py` and `delan_vs_vanilla.py`: shows directly where the extra structure of DeLaN pays off.*

## How to run

From the repo root:

```bash
python examples/lnn/lnn_vs_vanilla.py
python examples/lnn/lnn_vs_delan.py
```

Dependencies: `torch`, `numpy`, `matplotlib`, `scipy`.
Scripts import from `models/`, `systems/`, and `utils/` at the repo root.
